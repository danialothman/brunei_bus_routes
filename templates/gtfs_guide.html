Brunei Bus Routes · GTFS workbench

Data Entry Guide

You are turning photographed signboards and hand-traced routes into a machine-readable national timetable (GTFS) — the format used by Google Maps, Apple Maps and journey planners worldwide. This page tells you exactly what goes where. Keep it open in a second tab while you work in the workbench.

1 · The screen

The workbench has three columns:

Left — route list Every route, with a 4-segment progress meter next to its name: schedule · departures · operator · headsign. A segment turns green when that data is entered. Your goal is four green segments on every route. Click a route to work on it.
Middle — map Shows the selected route's line (blue) and stops (yellow). ✎ Edit route opens the drawing tools. Official routes are read-only — editing one automatically creates a copy under "My routes", which is the one you edit from then on.
Right — route form Everything about the selected route: schedule, stops list, names, operator, and the timing signboard photo you transcribe from. Every field saves automatically a moment after you stop typing — watch for Saved ✓.

2 · Per-route checklist

Work through one route at a time, top to bottom:

Step 1 — Identify the route

No. The public route number from the signboard, e.g. 01A, 20, 38.
Name A human-readable route name, e.g. BSB – Pekan Muara via Airport.
Description The fuller routing summary if known, e.g. BSB → Mall Gadong → Batu Bersurat: Kiulap — Jame'Asr — …. The reference list in docs/2016/reference/routes.txt has these for most routes — copy them in.
Color Optional hex color like #E91E63 if the route has an official color. Skip if unknown.
Operator The company that runs this route. Pick from the dropdown — if the company isn't listed yet, add it first under ⚙ Feed settings (see section 4). Leave on (default) if unknown.

Step 2 — Type & headsigns

Type loop / one-way: the bus goes round a circuit back to the start (e.g. BSB Circular).
out & back: the bus drives to a far terminal and returns the same way (e.g. BSB → Pekan Muara → BSB). Choose this and the export automatically produces both directions.
Headsign What the front of the bus displays going out, e.g. PEKAN MUARA.
Return (out & back only) The sign coming back, usually BSB.
hail & ride Tick if the route stops on demand anywhere along the road, not only at marked stops — common in Brunei.

Step 3 — Schedule (from the timing signboard)

The signboard photo appears below the form — zoom with − / + / Fit. Transcribe what it says:

Days Tick the days this timetable applies to. If weekends differ, + Add day-type schedule and give the weekend its own block — e.g. block 1 Mo–Fr, block 2 Sa–Su.
Every … min, from … to … Use when the signboard gives an interval ("every 30 minutes, 6:00am–8:00pm"). If the interval changes during the day (peak / off-peak), + time band for each window:
Every 10 min, 06:00–08:30 · Every 30 min, 08:30–16:00 · Every 15 min, 16:00–19:00
Exact departures Preferred whenever the signboard lists actual times. Type them into the dark board, separated by commas or new lines: 06:30, 07:00, 07:45, 09:15. When filled, this beats the interval — the export contains one real trip per departure.
Return departures (out & back only) If the signboard lists departure times from the far terminal separately, enter them here. Leave empty and the system estimates them (outbound time + run time).
run … min End-to-end journey time in minutes, if the signboard or local knowledge gives it. Leave on auto to estimate from the route's length.

Step 4 — Stops

The Stops section lists every stop in driving order — the order matters, it is the order passengers experience.

numbered chip Click to fly the map to that stop (a red ring flashes). Use it to verify each row is the stop you think it is.
code The public stop number if one exists on the physical signage.
name The stop name as signed, e.g. PBS OPP TERRACE HOTEL. (PBS/EBS prefixes are kept as-is.)
lat / lon Fine-tune coordinates by typing, or drag the stop on the map while editing. Bad values revert automatically.
↑ ↓ ✕ Reorder or remove. To add stops, use ✎ Edit route → + Stop on the map, with the 🚏 Official stops signboard panel open for reference.

While the map editor is open the list is live: dragging a stop updates its row instantly, but nothing is saved until you press Save in the toolbar. Exit without saving and everything reverts. Outside the editor, list edits save automatically as new versions.

Step 5 — Check your work

  • All four meter segments green in the route list.
  • Click a few stop chips — does the map land where the stop really is?
  • Run ✔ Validate (top bar). A warning like "stop is over 150 m from its route's line" means the stop or the line is drawn in the wrong place — fix whichever is wrong.

3 · What the signboards give you

🚏 Official stops panel Photos of the official stop-list signboards. Open it (top bar) while placing or renaming stops. It has a notes field per route — record discrepancies there (e.g. "poster lists a stop the trace doesn't have").
Timing signboard (in the form) Photos of the official departure-time boards. This is the source for Step 3. If a route has no timing photo, enter interval service from local knowledge and note it in the stops panel notes.

4 · Once per dataset — ⚙ Feed settings

Operators One row per bus company: name, website, phone. The first row is the default for routes with no explicit operator. Add all known companies here before assigning them per route.
Holidays Public holidays where service changes: pick the date and either no service or Sunday timetable. Add Hari Raya, National Day, etc.
Fare The flat fare, e.g. 1.00 BND.

6 · Golden rules

  1. Transcribe, don't invent. Enter what the signboard says. If you must estimate, leave a note in the 🚏 stops panel.
  2. Exact departures beat intervals. If the board lists times, type the times.
  3. Stop order is sacred — it must match driving order.
  4. Official routes are never edited directly — the ✎ button gives you a copy; that's intentional, the original stays as evidence.
  5. Validate before you stop for the day. Zero errors always; investigate warnings.
  6. Everything autosaves and is versioned — you can't destroy anything, so work boldly.
